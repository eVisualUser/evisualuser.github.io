Pretty Engine

Work in progress game engine for 2D games.
I am currently working on it to learn game engine development. It will end by making a game using it, probably an open-world 2D game.

For the creation part, coding must be done in C++ only, but with a simplified pipeline The editor manage all the artist part.
To create a game you need to duplicate the code of the engine and compile it yourself if you add some script, this is to allow you to customize the engine, and also the C++ scripting part need it. This allows the editor to have access to your custom Components and Entities.

For the organization you can script Entities that will work standalone, and you can use polymorphism on it. You can also create components that is more dynamic but have some performances and memory cost.

It is written in C++ and using CMake. I choose C++ because I wanted to create an engine that would train me to improve my skills in this language. By C++ being very important in Game Engine development, I found it necessary to be sure that I learn things applicable to the industry.

I use Go lang to write scripts that manage the engine sources and dependencies. It target to improve the ease of use of the engine by installing the dependencies and generating the files, and requiering only to have Go installed.

For the launcher I use Tauri, to have a solution easy to maintain and get great visuals, compared to most of native GUI/UI frameworks. And compared to Electron I get better performances and it's based on my favorite programming language: Rust.
